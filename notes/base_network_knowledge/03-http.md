# http

#### http

``` javascript
///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


http 是基于 请求<->响应 的方式完成通信
每一次通信都是由客户端向服务器发出请求，传递一些消息
再经过服务器处理后，响应给客户端一些消息


//-------------------------------------------------------------------------------------------------------------------//


1. 每次 请求<->响应 都是独立的，相互之间不干扰
( 该模式称为: 无状态歇息 )

2. 每次 请求<->响应 传递的消息都是 "字符串"
( 该字符串格式必须遵循http协议规定 )


///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
```

#### request

``` javascript
///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


POST /api/demo HTTP/1.1                             // 请求行: 
HOST: www.demo.com                                  // 请求头: 描述请求
Content-Type: application/json      # 请求体数据格式( 没有请求体， 则该字段无意义 )
User-Agent: Mozilla/5.0)            # 客户端信息
Origin: https://demo.com:8080       # 

{"username": "alpha"， "password": "123"}            // 请求体( 可省略 ): 必须和请求头空一行


//- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -//


// 请求行: 

请求方法 路径 参数 Hash

    - 请求方法:
    
        * 开发约定
            
            - Resfull 接口约定: GET POST PUT DELETE
            
            - GET & DELETE 没有请求体
    
    - 路径:        // 
    
    - 参数:        // 
    
    - Hash        // 


// 请求头

    - Content-Type
    
        * application/x-www-form-urlencoded         // 表示请求体的数据格式和url地址中参数的格式一样
        
        * application/json                          // 表示请求体的数据格式是 json
        
        * application/form-data                     // 文件上传一般用该格式
        
        * ...


// 请求体: 由于请求体格式的多样性， 服务器在分析请求体时可能无法知晓具体格式， 从而不知道如何解析请求体
          因此， 服务器要求在请求头中需附带属性 Content-Type 用于描述请求体使用的格式

    - 服务器识别
        
        * application/x-www-form-urlencoded
        
            key=val&key=value&key=value
        
        * application/json
        
            {"key": "value"，"key": "value"，"key": "value"}
        
        * application/form-data
        
            使用某种随机字符串作为属性之间的分隔符，通常用于文件上传


///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
```

``` javascript
///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


HTTP/1.1 200 OK                                     // 响应行
Content-Type: text/html;charset=utf-8               // 响应头
Server: BWS/1.1

<!DOCTYPE html>                                     // 响应体
<html lang="en">
<heaad>
  <title>Document</title>
</head>
<body>
    <h1>Hello World</h1>
</body>
</html>


//- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -//


// 响应行

协议版本 状态码

    - 协议版本: 
    
    - 状态码: 服务器对当前请求的表态      // 0~399 状态n码均为正常情况
    
        - 1**       // 信息           服务器收到请求，需要请求者继续进行操作
        
        - 2**       // 成功           操作成功接收并处理
        
        - 3**       // 重定向          需要进一步操作以完成请求
        
        - 4**       // 服务端错误       请求包含语法错误或无法完成请求  
        
        - 5**       // 服务器错误       服务器处理请求的过程中发生了错误


// 常见状态码

    - 200 ok                          一切正常
    
    - 301 Moved Permanently           资源永久重定向
    
    - 302 Found                       资源临时重定向
    
    - 304 Not Found                   资源临时重定向
    
    - 400 Not Modified                文档内容未被修改
    
    - 403 Forbidden                   服务器拒绝访问
    
    - 500 Internal Server Error       服务器内部错误


// 响应头

    - Content-Type
    
        * text/plain                纯文本，浏览器通常将响应体原封不动的显示到页面上
        
        * text/html                 html文档，浏览器通常会将响应体作为页面进行渲染
        
        * text/javascript           浏览器通常使用 js引擎 解析执行
        
        * text/css                  浏览器将作为 样式执行
        
        * image/jpg                 附件，浏览器看到这个类型，通常会触发下载功能
        
        * attachment
        
        * mime


// 响应体: 响应的主体内容


///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
```
